# SubLink — 产品需求文档（PRD）

> 项目类型：面向个人用户的静态工具站；推荐采用 Vue 2 + Element UI + GitHub Pages，以最小改动复用成熟转换前端。

## 背景与目标

- 一句话描述：把一个或多个代理订阅/节点链接转换为常见客户端可导入的订阅地址。
- 核心价值：在一个页面完成格式选择、规则选择、高级参数配置和链接复制。
- 成功指标：页面可用性 ≥ 99.5%；首屏静态资源在典型宽带下 3 秒内完成；关键生成流程成功率 ≥ 95%（可用后端前提下）。
- 商业模式：MVP 为免费开源工具，不包含账号、计费或广告。

## 功能需求

### MVP

- **P0** 作为普通用户，我要输入一个或多个链接并选择目标格式，以得到转换链接。验收：必填项缺失时阻止生成；合法输入能生成带编码参数的 `/sub` URL。
- **P0** 作为高级用户，我要选择或输入转换后端。验收：后端切换后请求 `/version` 并展示可用性结果。
- **P0** 作为维护者，我要在推送 `main` 后自动发布。验收：GitHub Actions 构建成功且 Pages URL 返回 HTTP 200。
- **P1** 作为用户，我要选择远程规则、包含/排除节点、UDP/TFO/Emoji 等参数。验收：选项正确进入生成 URL。
- **P1** 作为用户，我要复制长链接或生成短链接。验收：支持手动复制；短链失败时给出明确提示。
- **P2** 作为用户，我要跟随系统主题并手动切换明暗模式。

### V2

- 后端健康状态缓存与排序。
- PWA 离线壳、分享预设、可导入导出本地配置。
- 自动化单元测试和 Playwright E2E。

## 非功能需求

- 性能：压缩后静态资源可被 Pages CDN 缓存；Lighthouse Performance 目标 ≥ 80。
- 安全：不持久化订阅内容；外部链接使用 HTTPS；依赖定期扫描。
- 可访问性：目标 WCAG 2.1 AA，表单有标签、键盘可操作、明暗主题对比清晰。
- 兼容性：最新两个版本 Chrome、Edge、Firefox、Safari；375px–1440px 视口。

## 风险评估

| 风险 | 影响 | 概率 | 缓解 |
| --- | --- | --- | --- |
| 第三方转换后端不可用 | 高 | 中 | 支持多后端和用户自建地址，提供版本探测 |
| 第三方后端会接收订阅 URL | 高 | 中 | 页面明确数据流，允许填写自建后端 |
| Vue 2 生态进入维护期 | 中 | 高 | 锁定依赖；V2 评估 Vue 3 迁移 |
| 远程规则链接失效 | 中 | 中 | 保留默认规则并定期抽检 |

## 约束与假设

- GitHub Pages 仅托管静态前端，不承担转换计算。
- 转换、短链和配置托管能力依赖 `.env` 中配置的第三方服务。
- MVP 不引入账号、数据库、支付或自建 API。
