# SubLink — 测试计划

## 测试策略

- 测试金字塔目标：单元 60%、集成 25%、E2E 15%。
- V2 覆盖率目标：URL 构造单元测试 ≥ 85%，集成路径 ≥ 70%。
- PR：安装、构建、单元测试；合并 main：构建并部署；每日：后端 `/version` 冒烟测试。

## 测试数据

- 仅使用本地占位节点和公开测试 URL，不使用真实订阅凭据。
- 用固定表格覆盖单链接、多链接、Unicode、特殊字符、空值和超长值。
- 本项目无测试数据库，无需初始化或清理数据库。

## 集成测试

| 外部接口 | 场景 | 请求 | 预期 |
| --- | --- | --- | --- |
| `/version` | 后端正常 | GET | HTTP 200，页面展示版本信息 |
| `/version` | CORS/超时 | GET | 页面展示后端不可用，表单仍保留 |
| `/short` | 正常短链 | POST FormData | 返回短链并可复制 |
| `/short` | 服务端错误 | 非成功响应 | 展示服务消息或通用失败提示 |
| `/sub.php` | 配置上传 | POST 编码文本 | 返回远程配置 URL |

## E2E 关键旅程

```gherkin
场景: 生成 Clash 订阅链接
  假如 用户打开首页
  当 用户输入测试订阅并选择 Clash
  并且 点击“生成订阅链接”
  那么 “定制订阅”中出现以所选后端开头的 URL

场景: 自定义后端
  当 用户输入 https://backend.example
  那么 页面请求 /version
  并且 后续生成 URL 使用该 origin

场景: 移动端操作
  假如 视口宽度为 375px
  那么 表单没有水平溢出且主要按钮可点击
```

## 安全测试

- XSS：在文件名、include/exclude、后端字段输入 HTML/脚本，确认仅作为文本/URL 参数处理。
- CSRF：本站无自有会话；检查第三方 POST 不携带本站凭据。
- 注入：确认所有用户值编码后进入 query string。
- 供应链：运行 `yarn audit`/Dependabot；工作流 Action 固定主版本。
- 信息泄露：构建产物、日志和 localStorage 中不得出现测试订阅 URL。

## 可访问性测试

- Lighthouse 与 axe-core 扫描，目标 WCAG 2.1 AA。
- 键盘完成输入、下拉选择、生成、复制和主题切换。
- 明暗主题分别检查文本/控件对比度；200% 缩放无内容遮挡。
- NVDA/VoiceOver 抽检表单标签和错误提示。

## 边界与错误场景

1. 空链接或只含空格。
2. 多个链接混合 CRLF/LF。
3. 含 `&`、`#`、Unicode 的链接和文件名。
4. 后端返回 403、500、非文本版本或 CORS 拒绝。
5. 剪贴板 API 不可用、短链响应格式异常。
6. 远程配置链接失效或超过输入长度。

## 性能建议

- 用 Lighthouse 记录 FCP/LCP/总阻塞时间和资源体积。
- 检查 GitHub Pages 缓存与 gzip/brotli 响应。
- 大型 Three.js 背景对低端移动设备的影响应单独对比；若 LCP 超标则延迟或移除动画。

Jest/Vitest（纯函数）、Vue Test Utils（组件）、Playwright（E2E）、axe-core（无障碍）、Lighthouse（性能）、Dependabot（依赖）。
